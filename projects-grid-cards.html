source code snippet

The picture below is a code snippet of how I developed the School fees part of the Accounting - section. I designed the system from the feedback of numerous accountants in different schools. They expected different systems, but they all had some things in common. As you can see, I used java 8 and applied generics. I also included comments ( in yellow) for every code block.

ACADEMIC - SECTION'S INTERFACE 

each button on the screen enables you to access a specific page to perform  a specific academic work

ADD STUDENTS button

it opens this interface where a new student can be admitted into the school.  The page is partitioned. The left side is for admitting a single student. The right side is for admitting multiple students at the same time, by importing an excel sheet containing student data

EXAMINATION button

All things exam, are performed here.  Marks of students are intered or imported here, the system auto does the calculation, giving each student his respective class position, printing terminal report of students, and exporting the MASTER SHEET  of the exam.

SEARCH button

this button takes you to a page where you can find any student in the school, by entering a name of that student / his student ID.  From here, you can view a student's entire academic history. You can also view and edit a student's personal information from here

CLASSES button

Here,  you can select a class and add its subjects and subject teachers. You can also view the list of students found in that class, their personal profiles and exam histories.

TERM/SEMESTER button

This button takes you to a window that allows you to set the current term and semester. This term/semester information will appear on  students' terminal reports.  In Ghana, TERM is used for students in basic 6 and below, while SEMESTER is used for students above basic 6. 

PROMOTE STUDENTS button

at the end of the third term / second semester, students are promoted to the next higher class. This button takes you to a window to achieve that aim. In this window, you select the current class, then click on 'promote students' (shown in green). Then the software automatically moves all the students to the next class, sets the new term/ semester for each, and gives each student a new and empty exam report for this new class

BACKUP button

This button appears either in the academic - section or the accounting - section, depending on the recommendation of the school I built this software for. The backup button takes you to the backup window (shown above). The 'save backup' (red button), saves all the personal, academic and exam information of all the students to the selected backup location, with one click. The 'load backup' (green button), restores  the saved backup back into the software, also with just one click

ACCOUNTING
